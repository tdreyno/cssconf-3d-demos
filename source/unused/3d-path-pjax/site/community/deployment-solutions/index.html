Deployment Solutions

Middleman concerns itself only with creating static websites, however through extensions there are a variety of deployment solutions to aid putting your /build where it needs to go.

The highlighted extentions below allow middleman build to deploy using RSync, S3, Git, FTP, Github Pages, Heroku ( Statically ), Google Storage, Rackspace Storage, etc...

Please check out all Community Extensions

Middleman-Deploy

Deploy a middleman built site over rsync or to github pages.

# Activate deploy extension
activate :deploy do |deploy|
  deploy.method = :ftp
  deploy.host = "ftp.example.com"
  deploy.user = "tvaughan"
  deploy.password = "secret"
  deploy.path = "/srv/www/site"
end

Middleman-Sync

Synchronise your Middleman build to S3 and more

# Activate sync extension
activate :sync do |sync|
  sync.fog_provider = 'AWS'
  sync.fog_directory = 'bucket-name'
  sync.fog_region = 'bucket-region-name'
  sync.aws_access_key_id = 'super'
  sync.aws_secret_access_key = 'secret'
  sync.existing_remote_files = 'keep'
end

Shart

Deploy middleman to an S3 bucket without conflating deployment configuration with app configuration.

First, install shart.

$ gem install shart

Then create a Shartfile in the root of the middleman project.

# A Middleman Shartfile.
source './build'
target 'your-aws-bucket-name', {
  :provider               =>  'AWS',
  :aws_secret_access_key  =>  'your AWS credentials',
  :aws_access_key_id      =>  'your AWS credentials'
}

Finally, build middleman and shart it into an S3 bucket.

$ middleman build && shart
